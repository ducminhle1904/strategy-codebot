# Security Policy

## Supported Versions

The project is currently pre-1.0. Security fixes target the latest `main` branch until formal releases are cut.

## Reporting a Vulnerability

Open a private security advisory on GitHub when available, or contact the repository owner directly. Do not open public issues for exploitable vulnerabilities until a fix or mitigation is ready.

## Safety Boundary

This project does not place live trades, connect to broker accounts, or claim strategy profitability. Reports that bypass these boundaries are treated as security and safety issues.
